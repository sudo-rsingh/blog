How to modify the size of AWS volume

When customer base and company requirement increases, we have to increase the size of our servers also. This could be a daunting task as we don't want to break our servers.

23 July 2025

We can divide this work in 2 parts, first being that we allocate more space to our volume, then se expand our system's partition and our system's file system.

Part 1: Expanding / Allocating more space to our EC2 machine

Note: I am assuming that you are accustomed to using basic aws and can make or delete instances.

Step 1.1: Open storage pane in your instance summary, you can find it at the lower part of your aws console. It should be near of networking, memory etc.

Step 1.2: Click of volume ID, of the volume which you want to modify. This will open the volume managment pane. There again choose your volume.

Choosing the same thing twice seems a bit dumb, but what aws gods gave us we have to take it without questions.

Step 1.3: Now click on Action drop down menu, and choose modify from it. This will tak us to the modify pane where we can make changes to our current running volume.

Step 1.4: Now here we can choose how we want to modify the volume.

  • Size (GiB): Total size of the volume.
  • IOPS: Input output operations per second.
  • Thoughput: Actual data transmission rate.

This will conclude all the things we need to do in the console. In the next part we will be making changes from inside the ec2 machine.

Part 2: Expanding the Partition and filesystem of our EC2 machine

Step 2.1: Check the size of partition.

sudo lsblk

Step 2.2: Expand / Grow the partition size.

sudo growpart /dev/nvme0n1 1

Step 2.3: Check the filesystem size.

df -hT

Step 2.4: Resize the filesystem.

sudo resize2fs /dev.nvme0n1p1
